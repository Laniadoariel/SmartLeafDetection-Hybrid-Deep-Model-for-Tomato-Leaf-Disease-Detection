Step-by-Step Guide
0

Prerequisites & Setup

Create a virtual environment and install dependencies.

# Create venv (if not already done)
python3 -m venv venv
source venv/bin/activate

# Install all dependencies
pip install -r requirements.txt
requirements.txt includes: ultralytics, opencv-python, torch, torchvision, numpy, hypothesis, pytest, pyyaml
1

Run Tests — Verify Everything Works

Run the full test suite to confirm all 65 tests pass.

source venv/bin/activate
python3 -m pytest tests/ -v --tb=short

Expected output: 65 passed

2

Extract Leaf Annotations from Red-Marked Frames

Detect the red bounding boxes on annotated images, extract YOLO-format labels, and copy the clean originals into a training dataset.

python training/extract_annotations.py \
    --annotated 1.jpeg 2.jpeg 3.jpeg \
    --originals 1_original.jpeg 2_original.jpeg 3_original.jpeg \
    --output-dir training/dataset \
    --min-area 1000

This creates:

training/dataset/
  ├── images/          # Clean original frames
  │   ├── 1.jpeg
  │   ├── 2.jpeg
  │   └── 3.jpeg
  ├── labels/          # YOLO-format .txt annotations
  │   ├── 1.txt
  │   ├── 2.txt
  │   └── 3.txt
  ├── visualizations/  # Green bbox overlays for verification
  │   ├── 1_viz.jpeg
  │   ├── 2_viz.jpeg
  │   └── 3_viz.jpeg
  └── data.yaml        # YOLO dataset config
Check the visualizations! Open training/dataset/visualizations/ to verify the green boxes match the original red annotations.
3

Train YOLOv11 for Leaf Detection

Fine-tune a pretrained YOLOv11-nano model on the extracted leaf annotations.

python training/train_yolo_leaves.py \
    --data training/dataset/data.yaml \
    --model yolo11n.pt \
    --epochs 50 \
    --imgsz 640 \
    --batch 4

Output: yolo11_leaves.pt in the project root.

3 images is extremely small — this is proof-of-concept only. For production, you'd need 500+ annotated images. The model will overfit but demonstrates the training pipeline works.
4

Train ResNet50 for Disease Classification

Fine-tune ResNet50 on the PlantVillage dataset (10 tomato disease classes).

# Download PlantVillage dataset from Kaggle + train
python training/train_resnet50.py \
    --download \
    --data-dir training/plantvillage \
    --epochs 25 \
    --batch-size 32 \
    --device cpu

Output: resnet50_tomato.pt in the project root.

Kaggle API required for auto-download. Install with pip install kaggle and set up ~/.kaggle/kaggle.json. Or download manually from kaggle.com/datasets/emmarex/plantdisease.

The 10 disease classes:

Bacterial_spot, Early_blight, Late_blight, Leaf_Mold,
Septoria_leaf_spot, Spider_mites, Target_Spot,
Tomato_Yellow_Leaf_Curl_Virus, Tomato_mosaic_virus, healthy
5

(Optional) Train YOLOv11 for Plant Detection

If you have plant-level annotations, train a separate YOLO model for plant detection. This step requires your own annotated dataset with whole-plant bounding boxes.

# Same workflow as leaf training, but with plant annotations
# Output: yolo11_plants.pt
For now, the leaf detection model can serve double duty if your drone frames show individual plants clearly.
6

Run the Full Pipeline on a Drone Video

Once you have trained weights, run the pipeline on a real drone video.

# Create a config file
cat > pipeline_config.yaml << 'EOF'
video_path: "drone_flight.mp4"
plant_weights_path: "yolo11_plants.pt"
leaf_weights_path: "yolo11_leaves.pt"
classifier_weights_path: "resnet50_tomato.pt"
device: "cpu"
extraction_rate: 3
aggregation_window: 30
dual_threshold_enabled: true
output_format: "json"
output_path: "disease_report.json"
EOF
# Run the pipeline via Python
python3 -c "
from smart_leaf_detection.config import PipelineConfig
from smart_leaf_detection.pipeline import Pipeline

config = PipelineConfig.from_file('pipeline_config.yaml')
pipeline = Pipeline(config)
records = pipeline.run(
    video_path=config.video_path,
    srt_path='drone_flight.srt'  # optional GPS
)
print(f'Found {len(records)} diseased plants')
"

Output: disease_report.json with one record per diseased plant.

Architecture — Module Breakdown

🎬 Frame Extractor

Reads drone video and yields BGR frames. Supports every-N-frames or target-FPS extraction modes. Streaming — no disk writes.

OpenCV
smart_leaf_detection/frame_extractor.py

🌱 Plant Detector

Detects tomato plants in full video frames using YOLOv11. Returns bounding boxes with confidence scores.

YOLOv11Ultralytics
smart_leaf_detection/plant_detector.py

🔗 Plant Tracker

ByteTrack-style IOU tracker. Two-stage association (high + low confidence) assigns stable PlantIDs across frames.

ByteTrackIOU Matching
smart_leaf_detection/plant_tracker.py

✂️ ROI Cropper

Crops plant/leaf regions with configurable padding. Boundary-clamped to prevent out-of-frame access.

NumPy
smart_leaf_detection/roi_cropper.py

🍃 Leaf Detector

Detects individual leaves within plant ROIs using a second YOLOv11 model. Coordinates are relative to the plant crop.

YOLOv11Ultralytics
smart_leaf_detection/leaf_detector.py

🔗 Leaf Tracker

Per-plant ByteTrack IOU tracker. Assigns stable LeafIDs scoped to each PlantID.

ByteTrackIOU Matching
smart_leaf_detection/leaf_tracker.py

📐 Leaf Normalizer

Resizes leaf ROIs to 224×224, converts BGR→RGB, applies ImageNet mean/std normalization. Outputs a (1,3,224,224) tensor.

OpenCVPyTorch
smart_leaf_detection/leaf_normalizer.py

🔬 Disease Classifier

ResNet50 with custom FC head. Classifies normalized leaf tensors into 10 tomato disease classes via softmax.

ResNet50torchvision
smart_leaf_detection/disease_classifier.py

⏱️ Temporal Aggregator

Sliding window buffer per LeafID. Dual-threshold gating: mean probability + majority vote ratio for confident labels.

Sliding WindowDual Threshold
smart_leaf_detection/temporal_aggregator.py

🏥 Plant Status Engine

Merges leaf-level labels into plant-level status. Reports top-K diseases ranked by accumulated confidence.

Top-K Ranking
smart_leaf_detection/plant_status_engine.py

📍 GPS Associator

Parses DJI-style SRT telemetry files. Maps subtitle blocks to frame indices for approximate plant geolocation (±2-5m).

SRT ParserRegex
smart_leaf_detection/gps_associator.py

📊 Report Exporter

Exports disease records as JSON or CSV. One record per diseased plant with GPS, disease labels, and evidence metrics.

JSONCSV
smart_leaf_detection/report_exporter.py

Quick Start — Copy & Paste

🚀 Full workflow in one go

# 1. Setup
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# 2. Verify tests pass
python3 -m pytest tests/ -v --tb=short

# 3. Extract annotations from your red-marked frames
python training/extract_annotations.py \
    --annotated 1.jpeg 2.jpeg 3.jpeg \
    --originals 1_original.jpeg 2_original.jpeg 3_original.jpeg \
    --output-dir training/dataset

# 4. Train leaf detector (YOLOv11)
python training/train_yolo_leaves.py \
    --data training/dataset/data.yaml \
    --epochs 50

# 5. Train disease classifier (ResNet50)
python training/train_resnet50.py \
    --download --data-dir training/plantvillage \
    --epochs 25 --device cpu

# 6. Run pipeline on a drone video
python3 -c "
from smart_leaf_detection.config import PipelineConfig
from smart_leaf_detection.pipeline import Pipeline

config = PipelineConfig(
    video_path='your_drone_video.mp4',
    plant_weights_path='yolo11_plants.pt',
    leaf_weights_path='yolo11_leaves.pt',
    classifier_weights_path='resnet50_tomato.pt',
    device='cpu',
    output_path='disease_report.json',
)
pipeline = Pipeline(config)
records = pipeline.run('your_drone_video.mp4', srt_path='your_video.srt')
print(f'Detected {len(records)} diseased plants')
for r in records:
    print(f'  Plant {r.plant_id}: {r.disease_labels}')
"
